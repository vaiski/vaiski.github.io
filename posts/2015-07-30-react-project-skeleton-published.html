Kickstarting React.js Projects

TL;DR

React project skeleton published at Github. Clone it, use it, break it, tweak it - then, repeat.

If you're like me, you often find yourself playing with new tech and creating small applications or scripts for your own amusement. Usually, speed is the key when it comes to sudden inspiration. That's why React is my library of choice nowadays for projects from rapid prototyping to production-ready applications when it comes to single-page applications (SPAs).

(source: Facebook's Flux architecture)

Thanks to React, creating a SPA is fairly straightforward; your representation markup and associated event handlers live in a same file. Sprinkle some Flux actions and dispatchers on it and light up your stores and watch as the information drives nicely down your newly built one-way lane.

(source: Slashgear)

Introducing React Project Skeleton

Crafting the project itself, as much as I like it, usually slows me down even down to that point where my inspiration has flowed down the drain and I sit only with an empty project structure trying to remember what I actually wanted to do. Creating a project template was my first step towards DRYness in my hobby projects.

React project skeleton combines my favourite technologies in SPA development: ES6, React, Flux, Backbone, Gulp and Mocha. You can grab the template and start playing with it by running just a handful of commands (assuming you have Node.js/io.js installed): 

$ git clone -o react-project-skeleton -b master --single-branch \
      https://github.com/vaiski/react-project-skeleton.git MyApp
$ npm install -g gulp
$ npm install
$ gulp

The project builds and opens up a new browser to live preview your changes.
